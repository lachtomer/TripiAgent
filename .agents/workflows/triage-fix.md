# /triage-fix Workflow

**Goal:** Provide a deterministic, self‑contained process for any Antigravity agent to triage, fix, and report a failure (lint, unit test, E2E, or build).

### Execution Cycle (automated by the agent)
1. **Reproduce**
   - Run the failing command (`npm run lint`, `npm test`, `npm run test:e2e`, `npm run build`).
   - Capture stdout/stderr and store in a temp log file.
2. **Triage**
   - Classify the failure using the **Failure Classification** table from `AGENTS.md` (deterministic bug, flaky timing, external nondeterminism, env/config, test bug, unclear).
3. **Root‑Cause Hypothesis**
   - Draft a one‑sentence hypothesis with supporting evidence (file/line, stack trace, network request).
4. **Implement Minimal Safe Fix**
   - If classification = deterministic code bug → edit app code.
   - If flaky timing → replace arbitrary `wait`s with explicit readiness signals (e.g., `await page.waitForSelector('[data-testid="ready"]')`).
   - If external nondeterminism → add Playwright `page.route` interception with a fixture JSON.
   - If env/config → adjust `.env.example` or CI config.
   - If test bug → edit the test to assert user‑visible behavior using stable selectors.
5. **Run Targeted Test(s)**
   - Execute only the affected spec(s) or lint run.
   - If still failing, loop back to step 2 (max 2 iterations).
6. **Self‑Heal Safe Failures**
   - Run `npm run lint` and automatically apply mechanical fixes (unused vars, missing imports, type mismatches) using the **Safe‑Fix** rules in `AGENTS.md`.
7. **Run Full Validation**
   - `npm run lint && npm test && npm run test:e2e && npm run build`.
8. **Report**
   - Emit the **Report Format** defined in `AGENTS.md` (triage classification, root cause, files changed, fix, tests run, remaining failures, risk, final status).

**Automation hints for agents:**
- Use `run_command` to execute shell commands.
- Use `view_file`/`grep_search` to locate relevant code.
- Use `replace_file_content` or `multi_replace_file_content` for edits.
- Use `schedule` to wait for background tasks.
- Do **not** ask the user any question unless the classification lands in *Unclear / mixed cause* and a product decision is required.
