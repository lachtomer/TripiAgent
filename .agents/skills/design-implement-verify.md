# design-implement-verify Skill

**Purpose:** Guide agents through the full design‑understand → implementation → verification cycle for new features or changes.

**Workflow Integration:**
- Invoked by the `/dev-loop` workflow after the planning step.
- Handles: reading design docs, creating component scaffolds, wiring API routes, adding Zod schemas, and writing initial unit/E2E tests.

**Key Actions:**
1. **Read Design Docs** – `view_file` the relevant markdown (e.g., `DESIGN.md`).
2. **Generate Boilerplate** – use `run_command` to scaffold components (`npm run generate component …`) or create files via `write_to_file`.
3. **Implement Logic** – apply code edits with `replace_file_content`/`multi_replace_file_content`.
4. **Add Tests** – create or update test files under `test/` or `e2e/` using stable selectors.
5. **Run Targeted Checks** – execute only the new tests.
6. **Self‑Heal** – automatic lint/type fixes as defined in `triage-and-fix`.
7. **Report** – emit a concise status using the standard *Report Format*.

**Auto‑activation:** The skill is automatically called when a new `implementation_plan.md` is detected or when a developer adds a `TODO` in `task.md` that references a design item.
