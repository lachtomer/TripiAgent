# /dev-loop Workflow

**Goal:** Provide a repeatable development loop that guides agents through design, planning, implementation, verification, and pause points.

### Development Loop Steps (automated by the agent)
1. **Design / Understand** – read the task description, relevant design docs, and existing code. Summarize intent.
2. **Plan Implementation** – create or update `implementation_plan.md` (see `USAGE.md`). Include proposed file changes and verification plan.
3. **Implement in Grouped Batches** – agents edit code in logical groups (e.g., component, API, store) using the appropriate skill (`design-implement-verify`).
4. **Run Targeted Checks** – execute only the tests directly affected by the change (e.g., `npm test <spec>` or `npm run lint <file>`).
5. **Self‑Heal Safe Failures** – if lint or type errors appear, apply mechanical fixes automatically (imports, typings, unused vars).
6. **Run Broader Validation** – `npm run lint && npm test && npm run test:e2e && npm run build`.
7. **Pause for Human Decision** – the agent stops only when one of the following is true:
   * A manual UI test or exploratory check is required.
   * A product/UX decision needs a stakeholder input.
   * Credentials, secrets, or destructive actions are needed.
   * There is a high‑risk change (e.g., data migration).
8. **Summarize Status** – emit a concise status report and next action.

**Auto‑activation:** The workflow is triggered when a developer creates a new `implementation_plan.md` or adds a `TODO` in `task.md`. The agent watches these files and starts the loop without user prompting.

**Manual invocation:** Users can run `/dev-loop` at any time to force the loop to start.
