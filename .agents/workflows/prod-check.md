# /prod-check Workflow

**Goal:** Perform final production verification before a release.

### Steps (automated)
1. **Lint** – `npm run lint`.
2. **Unit Tests** – `npm test`.
3. **E2E Tests** – `npm run test:e2e`.
4. **Build** – `npm run build`.
5. **Deployment Smoke** – Deploy to a staging environment (Firebase/App Hosting) and run a minimal Playwright smoke test against the live URL.
6. **Report** – Emit the standard *Report Format* with a clear statement:
   - `READY FOR GO‑LIVE` only if all four steps succeed.
   - `NOT READY` with listed failures otherwise.

**Auto‑activation:** Triggered automatically after a successful `/dev-loop` that reaches step 6.

**Manual invocation:** Users may run `/prod-check` at any time to force a full verification.
