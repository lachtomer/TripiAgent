# Usage Guide for Antigravity Operating System (TripiAgent)

## Available Slash Commands
| Command | Description |
| ------- | ----------- |
| `/triage-fix` | Runs the full **triage‑fix** workflow: reproduces a failure, classifies it, hypothesises root cause, applies a minimal safe fix, runs targeted then full validation, and reports. |
| `/dev-loop`   | Starts the **development loop** workflow. Guides the agent through design → plan → implementation → verification → pause points. Auto‑triggers on new `implementation_plan.md` or `TODO` in `task.md`. |
| `/prod-check`| Executes the **production verification** workflow: lint → unit tests → E2E → build → deployment smoke, then emits a readiness report. |
| `/scaffold`  | (internal) scaffolds project folders. |
| `/trip-step` | (internal) runs a single TripiAgent playbook step. |
| `/verify`    | (internal) performs a full local verify + browser smoke.

## Skills (auto‑activated by workflows)
- `triage-and-fix` – classification, root‑cause analysis, mechanical fixes.
- `e2e-stabilization` – selector hygiene, deterministic waits, network mocking.
- `production-readiness` – env/config validation, CI pipeline checks, build readiness.
- `design-implement-verify` – design understanding, scaffold generation, implementation, test creation.

### When skills load automatically
- **During a workflow** the appropriate skill is invoked based on the current step (e.g., `/triage-fix` calls `triage-and-fix`).
- **File‑watch triggers**: when a new `implementation_plan.md` or a `TODO` in `task.md` appears, the `design-implement-verify` skill is auto‑started by the `/dev-loop` workflow.

### When you must invoke a workflow manually
- To start a full failure‑fix cycle: run `/triage-fix`.
- To begin a new feature or change: run `/dev-loop` (or simply add an `implementation_plan.md`/`TODO`).
- To verify the entire release candidate: run `/prod-check`.

## How to know which skill/workflow is active
- The agent posts a status line in the chat each time it switches context, e.g.
  `🛠️ Using skill: triage-and-fix`
- When a workflow starts you will see a header like
  `🚀 Workflow started: /dev-loop`
- The generated **Report** (see `AGENTS.md`) always includes the workflow name.

## Overriding routing / forcing a specific skill
If you need to run a skill without the surrounding workflow, use the slash command with the skill name as a suffix, e.g.:
```
/triage-fix --skill e2e-stabilization
```
(You can append `--skill <skill_name>` to any workflow command to force that skill.)

## Permissions – what to set to **Allow** in Antigravity UI
- **File read/write** for the entire project directory (`c:\TripiAgent\`) – allow auto‑approval for all non‑destructive edits.
- **Command execution** – allow `npm` and `git` commands in the project root.
- **Network access** – allow outbound HTTP requests for fetching fixtures *only* when the skill `e2e-stabilization` runs (auto‑approved).
- **Disallow** (manual approval required):
  * Deleting files or directories.
  * Modifying `.env` files containing secrets.
  * Changing business logic that affects user‑facing behavior.
  * Deployments to production environments.

> **Note:** Most safe install/access requests (npm package installs, reading files, running test commands) will be auto‑approved once the above permissions are set.
